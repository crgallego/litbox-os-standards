# Git Standard

Small commits, meaningful messages, PR summaries reference decision records.

## Commit Practice

Commits should be small enough to review and named for the behavior or standard they change.

Prefer messages that explain the durable change:
- `Add repository adoption contract`
- `Clarify changelog migration notes`
- `Record decision-log requirements`

Avoid messages that only describe mechanical edits:
- `Update files`
- `Fix stuff`
- `Changes`

## Pull Request Practice

Pull requests should include:
- Summary of what changed
- Standards, templates, or examples affected
- Validation performed
- Changelog entry reference
- Decision record reference when applicable
- Product Council recommendation when applicable

## Branch Practice

Use short, descriptive branch names:
- `standards/repository-contract`
- `docs/changelog-governance`
- `templates/decision-log`
