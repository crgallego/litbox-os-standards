# Discovery Standard

Classify discoveries as: Implementation, Architectural, Product, Customer, Philosophical.

## Discovery Types

- Implementation: What we learned while building or operating the system
- Architectural: What we learned about structure, interfaces, boundaries, or dependencies
- Product: What we learned about the product's shape, value, or roadmap
- Customer: What we learned about user needs, language, pain, or behavior
- Philosophical: What we learned about how Litbox OS should think or operate

## Discovery Requirements

Each discovery should include:
- Classification
- Source or trigger
- What changed in our understanding
- Implication for current work
- Whether it should become a standard, decision record, product council item, or changelog note

## Promotion Rule

If a discovery changes how more than one repository should operate, propose a standards update here.
