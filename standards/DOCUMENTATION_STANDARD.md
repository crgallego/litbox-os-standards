# Documentation Standard

Document decisions, assumptions, tradeoffs, and rationale—not only outcomes.

## Documentation Should Preserve

- What changed
- Why it changed
- What was considered and rejected
- What remains uncertain
- How future contributors should revisit the decision

## Documentation Should Avoid

- Retrofitting certainty after the fact
- Repeating shared standards locally
- Hiding meaningful tradeoffs behind polished summaries
- Treating implementation details as separate from product and narrative learning

## Discovery First

When a change teaches something, record the discovery. A useful discovery is more valuable than a heroic-sounding story.
