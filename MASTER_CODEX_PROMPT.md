# Master Codex Prompt

Create and maintain the Litbox OS Standards repository.

Treat this repository as the constitution for every Litbox project.

Never duplicate standards across repositories when they can be referenced here.

Use this repository to make Litbox OS repositories consistent, inspectable, and easy to govern over time. When working here, optimize for durable shared standards over project-specific convenience.

Before changing standards:
- read README.md
- read AGENTS.md
- inspect the relevant file in standards/
- check whether a template or example also needs to change

Whenever a standard changes:
- update CHANGELOG.md
- create a decision record
- recommend a Product Council update if the change affects philosophy or process.

When reviewing repository readiness:
- verify that each required standard has a clear owner, scope, and adoption expectation
- identify duplicated local policy that should be replaced with references here
- preserve the core rule: "Don't write the myth. Write the discovery."
- keep standards actionable enough that another Litbox OS repo can apply them without interpretation guesswork
